Spot the Vulnerability

Hard

This snippet renders a comment straight from user input. Name the vulnerability class (one word, e.g. XSS, SQLI, CSRF).

app.get('/comment', (req, res) => { const c = req.query.text; res.send('<div>' + c + '</div>'); });
User input is concatenated directly into HTML with no escaping — what runs in the victim's browser?
03 · fundamentals

Core concepts, made visible

Hover the cards. The fundamentals I build every project against.

C

Confidentiality

Only the people who should see the data, can. Enforced through encryption and access control.

I

Integrity

Data isn't altered in transit or at rest without detection. Hashing and signatures prove it.

A

Availability

Systems stay up for the people who legitimately need them — resilient against failure and DDoS.

🎣

Phishing

Deceptive messages trick users into handing over credentials or clicking malicious links.
📝

XSS

Unsanitized input lets an attacker run script in another user's browser session.
🗄️

SQL Injection

Malicious input alters a database query, exposing or corrupting data it shouldn't touch.
🕵️

Man-in-the-Middle

An attacker secretly intercepts traffic between two parties who believe they're talking directly.
💥

DDoS

A flood of traffic from many sources overwhelms a service until legitimate users can't reach it.
🎭

Social Engineering

Manipulating people, not systems — the weakest link is often trust, not technology.
